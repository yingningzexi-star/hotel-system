# 辅助文档与设计资产存储规范

在协助用户生成临时总结、答辩汇报、发言模板、设计预览报告等 Markdown 说明文档或非代码设计资产时，请遵循以下规范：

1. **就近存储原则**：不要将上述辅助总结或汇报性质的文档保存在系统默认的临时缓存区（例如 C 盘的 App Data 目录）。应当直接在用户当前项目工作区根目录的 `docs/` 文件夹下创建并保存这些文件。
2. **设计资产内聚原则**：将设计生成的 HTML 页面（如幻灯片）及相关的图片、截图等媒体资产，保存在项目工作区根目录下的 `designs/<project-name>/` 文件夹下。
3. **自包含与相对链接**：在 `docs/` 目录下的 Markdown 说明文档中，对于 `designs/` 目录下多媒体资产的引用，应当拷贝一份到项目内，并优先使用相对路径（例如 `../designs/<project-name>/<asset-name>`），以保证项目整体搬迁时资产引用不失效，实现整个说明体系的自包含，方便用户在编辑器中随时查阅及清理。
